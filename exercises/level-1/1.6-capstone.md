# Exercise 1.6: Capstone -- The Complete Spec

## Scenario Context

You've done the work: problem statement, discovery, spec drafting, requirements sharpening, and validation. Now it's time to pull it all together into a polished, ship-ready spec.

Maya has scheduled a formal spec review meeting for next Wednesday. The attendees: Maya (Engineering Lead), David (Data Scientist), Priya (CSM), and your Engineering Manager, who hasn't seen the spec yet. Your EM will be evaluating whether this spec is clear enough to plan sprints from.

This is the moment of truth for your spec. It needs to stand on its own.

---

## Template Exercise

**Deliverable:** A final, polished Smart Dependencies spec that could be handed to an engineering team and used to plan and build the feature. Plus a self-assessment using the capstone rubric.

### Steps

1. **Final polish pass.** Read your spec end-to-end one more time. Check for:
   - Consistency: Do the requirements match the problem statement? Do the success criteria map to the requirements?
   - Completeness: Is every section substantive? (No placeholder text should remain.)
   - Clarity: Read each requirement out loud. Would a new engineer understand it?
   - Conciseness: Cut any sentence that doesn't earn its place. Specs should be comprehensive but not verbose.

2. **Verify cross-references.** Make sure:
   - Your constraints are reflected in the requirements (e.g., if there's a schema freeze constraint, no requirement should assume a schema change)
   - Your success criteria are achievable given the requirements
   - Your Won't Haves don't contradict anything in the Must Haves

3. **Update the header:** Set status to "Approved" and version to "1.0" (or increment if you've already hit 1.0).

4. **Write a 2-sentence spec summary** at the top of the document, right below the header. This is for busy stakeholders who need the gist without reading the full spec.

5. **Score yourself** using the [Level 1 Capstone Rubric](../../rubrics/level-1-capstone-rubric.md). Be honest. For each criterion:
   - Rate yourself 1-5
   - Write a brief justification for your score
   - If you scored below a 4 on anything, write one specific thing you'd do differently

   Save your self-assessment in `my-work/level-1/capstone-self-assessment.md`.

6. **Final commit:**
   ```
   git add my-work/level-1/
   git commit -m "Level 1 Capstone: final Smart Dependencies spec + self-assessment"
   ```

### What "Done" Looks Like

Your completed `my-work/level-1/` folder should contain:
- `initial-reactions.md` -- your first thoughts on the CEO's request
- `smart-dependencies-problem-statement.md` -- the problem statement
- `smart-dependencies-discovery.md` -- the discovery document
- `smart-dependencies-spec.md` -- the complete, polished spec
- `capstone-self-assessment.md` -- your rubric-based self-assessment
